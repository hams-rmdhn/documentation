**Bambu Runcing Security** is a simple WordPress security plugin designed to protect your site from various online threats. It offers user-friendly and effective security settings, making it easy to secure your website without requiring advanced technical skills. Focused on implementing best security practices, this plugin helps minimize risks and ensures your site's integrity.  

Key features include disabling XML-RPC, hiding the WordPress version, and deactivating the file editor to block common vulnerabilities. It also limits login attempts to prevent brute force attacks, enforces HTTPS connections, sets robust security headers, and allows custom redirects after login or registration. With its lightweight design and flexible options, Bambu Runcing Security is the perfect solution for maintaining a safe and secure WordPress site.

## Configuration

All of the configuration below is working without any file change in your .htaccess, wp-config, or any other files.

- `File Editor`: Disables the built-in WordPress file editor for themes and plugins, preventing unauthorized code modifications.
- `Always HTTPS`: Forces all connections to use HTTPS, ensuring data is transmitted securely between users and your site.
- `XML RPC`: Disables XML-RPC functionality, protecting your site from potential brute force attacks and unauthorized access attempts.
- `Security Headers`: Adds essential security headers like Content-Security-Policy, X-Content-Type-Options, and others to safeguard your site from common vulnerabilities.
- `Limit Login Attempts`: Restricts the number of login attempts to prevent brute force attacks on your login page.
- `Hide WordPress Version`: Removes WordPress version details from the frontend, backend, and source code to reduce the risk of targeted attacks.
- `Hide WordPress Footer Credit`: Removes or the default WordPress footer credit text in the admin panel.
- `Hide WordPress Version Credit`: Removes or WordPress version information in footer credit text in the admin panel.
- `Admin Bar`: Customizes or disables the admin bar for specific user roles to declutter the interface and improve security.
- `WordPress Logo`: Removes the WordPress logo from the admin bar, providing a more personalized branding experience.
- `Update Bar`: Hides the update notifications bar in the dashboard for non-administrator users.
- `Comment Bar`: Removes the comments icon from the admin bar for a cleaner interface.
- `New Content Bar`: Hides the "New" menu from the admin bar, preventing unauthorized content creation.
- `Redirection After Login`: Allows custom redirects after user login, improving user flow and navigation.
- `Redirection After Registration`: Enables custom redirects after user registration, streamlining the onboarding process.
